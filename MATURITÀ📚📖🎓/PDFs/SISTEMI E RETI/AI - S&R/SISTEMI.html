UdA n. 1: Modulo di Consolidamento e Fondamenti delle Architetture di Rete

1.1 Infrastruttura e Livelli Protocollari: ISO/OSI e TCP/IP

L'architettura delle reti moderne si fonda sulla scomposizione in livelli protocollari. Il Modello ISO/OSI (strutturato su 7 livelli logici isolati) funge da riferimento teorico, laddove lo Stack TCP/IP (composto da 4 livelli operativi) costituisce l'implementazione fisica e ingegneristica dell'infrastruttura Internet globale.

Al livello Network / Internet, l'indirizzamento IP garantisce l'identificazione univoca degli host. Al livello Transport, la comunicazione end-to-end viene gestita da due protocolli divergenti:

  • TCP (Transmission Control Protocol): Orientato alla connessione, affidabile, implementa il meccanismo del Three-Way Handshake (SYN, SYN-ACK, ACK), il controllo di flusso (finestra scorrevole) e la gestione della congestione. Garantisce la consegna ordinata e priva di errori dei segmenti.
  • UDP (User Datagram Protocol): Non orientato alla connessione, non affidabile, privo di riscontri (ACK). Ottimizzato per la massima velocità e il basso overhead, ideale per streaming multimediale in tempo reale e query DNS.
Concetti Chiave: Socket Architecture La Socket rappresenta l'astrazione software che identifica univocamente un punto di terminazione di una comunicazione di rete. Essa è definita rigorosamente dalla combinazione logica: Socket = Indirizzo IP + Porta Trasporto (es. 192.168.1.50:443 per una sessione HTTPS).

1.2 Configurazione e Indirizzamento: DHCP e DNS

L'assegnazione dei parametri IP all'interno di un dominio locale avviene dinamicamente mediante il protocollo DHCP (Dynamic Host Configuration Protocol), operante a livello applicativo su porte UDP 67 e 68. Il ciclo d'interazione si articola in quattro fasi distinte (DORA):

  1. DHCP Discover: Il client invia un frame in broadcast totale (255.255.255.255) per individuare i server attivi.
  2. DHCP Offer: I server rispondono in unicast/broadcast offrendo una configurazione IP disponibile (IP, Subnet, Gateway, DNS).
  3. DHCP Request: Il client richiede formalmente i parametri offerti dal server selezionato, notificando gli altri in broadcast.
  4. DHCP Acknowledge (ACK): Il server conferma l'assegnazione e fissa il tempo di locazione (Lease Time).

Il sistema DNS (Domain Name System) mappa in modo distribuito e gerarchico i nomi di dominio leggibili (es. www.esempio.it) in indirizzi IP numerici instradabili, sfruttando record di tipo A (IPv4), AAAA (IPv6), CNAME (alias) e MX (server di posta).

1.3 Traduzione degli Indirizzi: NAT e PAT

Per arginare l'esaurimento dello spazio di indirizzamento IPv4, le tecniche di NAT (Network Address Translation) e PAT (Port Address Translation) consentono di mappare un intero blocco di indirizzi IP privati (non instradabili su Internet, definiti dalla RFC 1918) dietro un unico o un ristretto pool di indirizzi IP pubblici operanti sul router perimetrale. Nel PAT, il router riscrive l'intestazione del pacchetto modificando l'IP sorgente e associando ad ogni host interno una porta sorgente TCP/UDP univoca generata dinamicamente, mantenendo una tabella di traduzione di stato internamente.

Technical Vocabulary & CLIL Integration Lease Time: The specific duration for which a DHCP server allocates an IP address to a client before it expires or requires renewal.
Three-Way Handshake: The reliable synchronization process used by TCP to establish a connection using SYN, SYN-ACK, and ACK flags.
Iterative Query: A DNS resolution process where the local resolver queries multiple authoritative nameservers step-by-step down the domain hierarchy.

UdA n. 2: Funzionalità e Caratteristiche dei Principali Servizi di Rete

2.1 Architetture Software dell'Application Layer

I servizi di rete operano seguendo due paradigmi distribuiti principali:

  • Client-Server: Struttura centralizzata asimmetrica. Un host sempre attivo (Server) risponde in ascolto su porte note alle richieste asincrone inviate da molteplici nodi client (es. HTTP su 80, HTTPS su 443).
  • Peer-to-Peer (P2P): Architettura simmetrica in cui ogni nodo (Peer) agisce contemporaneamente sia da client sia da server, condividendo porzioni di risorse computazionali e file system senza intermediari centralizzati.

2.2 Studio Analitico dei Protocolli Applicativi

Telnet e SSH

Telnet (porta TCP 23) consente l'avvio di sessioni terminali remote trasmettendo le credenziali e i comandi in chiaro, esponendo il sistema ad attacchi di intercettazione passiva (sniffing). SSH (Secure Shell) (porta TCP 22) sostituisce interamente Telnet implementando una cifratura asimmetrica per la fase di autenticazione e una cifratura simmetrica per la sessione di dati, garantendo riservatezza e integrità crittografica.

FTP (File Transfer Protocol)

Operante su porte TCP 20 (Dati) e 21 (Controllo). Si articola in due modalità:

  • Modalità Attiva: Il client si connette alla porta 21 del server e comunica una propria porta dinamica su cui il server dovrà avviare la connessione dati partendo dalla sua porta 20 (spesso bloccato dai firewall del client).
  • Modalità Passiva: Il client avvia entrambe le connessioni; il server indica al client su quale porta effimera aprirà il canale dati, superando i limiti restrittivi dei firewall periferici.

HTTP, WWW e la transizione a HTTPS

Il protocollo HTTP (HyperText Transfer Protocol) è di tipo stateless (privo di memoria di stato). I messaggi si dividono in Request (caratterizzate dai metodi GET, POST, PUT, DELETE) e Response (identificate da codici di stato strutturati, es. 200 OK, 404 Not Found, 500 Internal Server Error). La sua evoluzione sicura, HTTPS, incapsula il traffico HTTP classico all'interno di un canale cifrato governato dal protocollo TLS (Transport Layer Security), garantendo l'autenticazione del server tramite certificati X.509 e l'inviolabilità dei dati in transito.

Architettura della Posta Elettronica

Il trasferimento dei messaggi tra server di posta (MTA) è affidato esclusivamente al protocollo SMTP (porta TCP 25 o 587 con TLS). Per il recupero e lo scaricamento locale dei messaggi da parte del client finale (MUA), si impiegano due standard antitetici:

  • POP3 (Post Office Protocol v3): Porta TCP 110. Scarica localmente i messaggi cancellandoli di default dal server, limitando la sincronizzazione multi-dispositivo.
  • IMAP4 (Internet Message Access Protocol v4): Porta TCP 143. Mantiene i messaggi e le cartelle centralizzati sul server, consentendo una sincronizzazione costante e simultanea da più client remoti.

Protocolli Multimediali e IoT

Per la trasmissione di flussi multimediali in tempo reale, si adoperano i protocolli RTSP (Real-Time Streaming Protocol) per il controllo della riproduzione e RTP (Real-Time Transport Protocol) su base UDP per il trasporto effettivo dei pacchetti dati. In ambito IoT (Internet of Things), domina il protocollo ultraleggero MQTT (Message Queuing Telemetry Transport), basato su un paradigma di tipo Publish/Subscribe mediato da un entità centrale detta Broker, minimizzando il consumo di banda e l'energia dei sensori.

Technical Vocabulary & CLIL Integration Stateless Protocol: A communications protocol that treats each request as an independent transaction, completely unrelated to any previous request.
Publish/Subscribe: A messaging pattern where senders (publishers) categorize messages into topics without knowing who the specific receivers (subscribers) are.

UdA n. 3: Virtual LAN (VLAN)

3.1 Tecnologie di Livello 2 e Livello 3

Lo standard Ethernet (IEEE 802.3) opera al livello Data Link, governando la trasmissione dei frame basandosi sugli indirizzi fisici memorizzati nelle schede di rete (MAC Address, a 48 bit). Gli Switch unmanaged inoltrano i frame in modo trasparente popolando autonomamente una tabella interna (MAC Address Table o CAM Table) tramite l'ispezione dei MAC sorgente. Gli Switch managed consentono la configurazione avanzata delle porte, la gestione della sicurezza e la segmentazione in reti locali virtuali. I moderni Layer 3 Switch uniscono la velocità di commutazione hardware dello switch di livello 2 alle capacità di instradamento logico IP tipiche di un router di livello 3.

3.2 Architettura delle VLAN e Standard di Tagging IEEE 802.1Q

Una VLAN (Virtual Local Area Network) consiste in una segmentazione logica di una rete fisica locale di livello 2. Essa isola nativamente i domini di broadcast senza richiedere modifiche al cablaggio hardware, incrementando la sicurezza interna, ottimizzando le prestazioni di banda e semplificando la gestione amministrativa degli host.

Le porte di uno switch managed possono essere configurate secondo due modalità operative:

  • Access Port: La porta appartiene stabilmente ad una singola VLAN specifica. Il traffico viaggia in modo tradizionale senza alcuna modifica, poichè i dispositivi terminali (PC, stampanti) non sono a conoscenza dell'esistenza delle VLAN.
  • Trunk Port: Configurazione di collegamento ad alta capacità (generalmente tra switch o tra switch e router) che consente il transito simultaneo dei frame appartenenti a molteplici VLAN distinte sullo stesso canale fisico.

Per veicolare il traffico di VLAN differenti su un unico link Trunk senza ambiguità, interviene lo standard internazionale IEEE 802.1Q. Questo protocollo inserisce un'intestazione di 4 byte (Tag) all'interno del frame Ethernet originario, posizionato immediatamente dopo il campo MAC Sorgente.

Concetti Chiave: Struttura del Tag 802.1Q (4 Byte)
  • TPID (Tag Protocol Identifier - 16 bit): Fissato al valore esadecimale 0x8100 per identificare la presenza del frame modificato secondo lo standard 802.1Q.
  • TCI (Tag Control Information): Sotto-strutturato a sua volta in:
    - PCP (Priority Code Point - 3 bit): Utilizzato per la gestione della qualità del servizio (QoS / Class of Service).
    - DEI (Drop Eligible Indicator - 1 bit): Indica la priorità di scarto del frame in caso di congestione di rete.
    - VID (VLAN Identifier - 12 bit): Il valore numerico effettivo che specifica la VLAN di appartenenza (da 1 a 4094).

UdA n. 4: Tecniche Crittografiche Applicate alla Protezione dei Sistemi e delle Reti

4.1 Requisiti di Internet Security: Il Modello CIA Triad

La sicurezza informatica poggia sul modello a tre pilastri denominato CIA Triad:

  • Riservatezza (Confidentiality): Garantire che i dati siano accessibili solo alle entità legittimamente autorizzate, tramite l'ausilio di algoritmi crittografici.
  • Integrità (Integrity): Assicurare che le informazioni non subiscano modifiche arbitrarie, manomissioni o alterazioni dolose durante la trasmissione o lo stoccaggio, verificabile tramite funzioni di hash.
  • Disponibilità (Availability): Permettere agli utenti autorizzati l'accesso continuo e privo di interruzioni alle risorse e ai servizi aziendali, contrastando guasti hardware e attacchi DDoS.

A questi si aggiungono l'Autenticazione (verifica certa dell'identità degli interlocutori) e il Non Ripudio (impossibilità per un'entità di negare l'invio o la ricezione di una transazione eseguita, implementato via Firma Digitale).

4.2 Fondamenti Matematici e Teorici della Crittografia

Un sistema crittografico è formalizzato dalla quintupla (P, C, K, E, D), dove P è l'insieme del testo in chiaro (plaintext), C il testo cifrato (ciphertext), K lo spazio delle chiavi, E l'algoritmo di cifratura e D l'algoritmo di decifratura.

Il principio cardine della sicurezza moderna è enunciato dal Teorema di Kerckhoffs: la sicurezza di un cifrario deve dipendere esclusivamente dalla segretezza della chiave crittografica, e non dalla segretezza dell'algoritmo matematico utilizzato, che si assume essere di pubblico dominio. Claude Shannon ha successivamente integrato questo approccio introducendo i concetti fondamentali di:

  • Confusione: Rendere la relazione statistica tra il testo cifrato e la chiave il più complessa e non lineare possibile.
  • Diffusione: Distribuire la struttura statistica del testo in chiaro su tutto il testo cifrato (se un singolo bit del testo in chiaro muta, circa la metà dei bit del testo cifrato deve variare radicalmente - Effetto Valanga).

L'unico cifrario perfetto a sicurezza incondizionata (inviolabile anche con potenza computazionale infinita) è il One-Time Pad (OTP), a patto che la chiave sia generata in modo puramente casuale, sia lunga quanto il messaggio da cifrare, sia utilizzata una sola volta e mantenuta segreta.

4.3 Tassonomia dei Cifratori Storici e Moderni

I cifratori si dividono storicamente in Sostituzione (monoalfabetici come il Cifrario di Cesare, o polialfabetici basati su matrici come il Cifrario di Vigenère) e Trasposizione (riarrangiamento geometrico delle posizioni dei caratteri originali). I sistemi moderni si classificano invece in:

  • Crittografia Simmetrica (Chiave Privata Condivisa): Unica chiave per cifrare e decifrare. Estremamente veloce ed efficiente su grandi moli di dati. Il DES (Data Encryption Standard), operante a blocchi di 64 bit con chiave effettiva di soli 56 bit, è oggi obsoleto per vulnerabilità intrinseche agli attacchi a forza bruta. Il suo successore temporaneo Triple-DES (3DES) esegue tre iterazioni consecutive di cifratura/decifratura aumentando la lunghezza della chiave, ma risulta computazionalmente pesante rispetto al moderno standard AES (Advanced Encryption Standard).
  • Crittografia Asimmetrica (Chiave Pubblica/Privata): Coppia di chiavi matematicamente correlate. La chiave pubblica è accessibile a chiunque e serve a cifrare; la chiave privata è strettamente segreta ed è l'unica in grado di decifrare. Il pilastro fondamentale è l'algoritmo RSA (dai creatori Rivest, Shamir, Adleman), la cui sicurezza computazionale si basa sull'intrinseca difficoltà matematica della fattorizzazione in fattori primi di grandi numeri interi (Problema del Logaritmo Discreto).
Modello Matematico dell'Algoritmo RSA 1. Generazione delle chiavi:
- Scegliere due numeri primi di grandi dimensioni p e q.
- Calcolare il modulo n = p × q e la funzione d'Eulero φ(n) = (p - 1)(q - 1).
- Selezionare un esponente pubblico e tale che sia coprimo con φ(n) e 1 < e < φ(n).
- Calcolare l'esponente privato d come l'inverso moltiplicativo di e modulo φ(n), ovvero: d × e ≡ 1 (mod φ(n)).
- Chiave Pubblica = (e, n); Chiave Privata = (d, n).
2. Cifratura del messaggio m: c = me (mod n).
3. Decifratura del testo cifrato c: m = cd (mod n).

Per superare i limiti di lentezza computazionale dell'algoritmo RSA e risolvere il problema della distribuzione sicura della chiave simmetrica, si utilizza la Crittografia Ibrida. Le entità scambiano in modo sicuro una chiave simmetrica temporanea (Chiave di Sessione) utilizzando l'algoritmo di scambio chiavi di Diffie-Hellman, dopodiché cifrano l'intera comunicazione dati tramite un cifrario simmetrico rapido (AES).

L'integrità del messaggio è garantita dalle funzioni di Hash (algoritmi deterministici non invertibili a senso unico che generano un'impronta digitale a lunghezza fissa, come SHA-256). La Firma Digitale si ottiene cifrando l'hash del messaggio con la propria chiave privata, consentendo a chiunque possieda la chiave pubblica del mittente di verificarne l'autenticità e l'integrità. I Certificati Digitali X.509 emessi dalle Certification Authority (CA) legano legalmente una chiave pubblica alla reale identità di un soggetto, neutralizzando gli attacchi di tipo Man-in-the-Middle.

UdA n. 5: Tecniche di Filtraggio del Traffico di Rete (Firewall)

5.1 Analisi dei Rischi nei Sistemi Informativi

La protezione di un perimetro informatico impone l'identificazione sistematica degli asset (risorse fisiche come server e switch, e risorse logiche come database e sorgenti software). Le minacce si classificano in ambientali/naturali (alluvioni, incendi), umane involontarie (errori di configurazione dei sistemisti, distrazione dei dipendenti) e umane intenzionali (attacchi malware, esfiltrazione dati da parte di cracker). Le contromisure includono policy di sicurezza, sistemi di backup ridondati e difese perimetrali hardware.

5.2 Architettura dei Firewall e Categorie di Filtraggio

Il Firewall rappresenta il componente cardine della sicurezza perimetrale, incaricato di ispezionare e filtrare il traffico dati in transito tra reti a diverso livello di fiducia (es. Internet esterna e LAN aziendale interna). I sistemi di filtraggio si dividono in:

  • Packet Filtering Stateless: Analizza i singoli pacchetti in modo isolato, confrontandone i campi dell'intestazione (IP sorgente/destinazione, porte TCP/UDP, flag) con una lista di regole statiche (ACL), senza mantenere traccia delle sessioni attive complessive.
  • Packet Filtering Stateful (Stateful Inspection): Mantiene una tabella di stato interna (State Table) che traccia attivamente tutte le connessioni stabilite. Il firewall permette il transito dei pacchetti in ingresso solo se appartengono a una sessione legittima avviata precedentemente dall'interno della rete, innalzando drasticamente il livello di protezione.
  • Application Level Firewall (Proxy Server): Agisce al livello applicazione, interrompendo la connessione diretta tra client e server estraneo. Il Proxy rigenera interamente le richieste, ispezionando il contenuto effettivo dei payload (es. blocco di URL specifici, analisi di codice malevolo in transito), offrendo il massimo controllo a scapito delle prestazioni velocistiche.

5.3 Sicurezza Perimetrale: ACL e Zone Demilitarizzate (DMZ)

Le ACL (Access Control List) implementano il filtraggio sul router/firewall. Si dividono in Cisco IOS in:

  • ACL Standard (1-99): Filtrano il traffico basandosi esclusivamente sull'indirizzo IP Sorgente. Vanno applicate il più vicino possibile alla destinazione.
  • ACL Estese (100-199): Filtrano valutando IP Sorgente, IP Destinazione, Tipo di Protocollo (IP, TCP, UDP, ICMP) e Numero di Porta Trasporto. Vanno posizionate il più vicino possibile alla sorgente del traffico.

Per esporre servizi pubblici (Web Server, Mail Server) senza compromettere la sicurezza della LAN interna privata, si progetta una DMZ (Demilitarized Zone). Le architetture principali sono:

  • DMZ a vicolo cieco (Three-Legged Firewall): Un unico firewall fisico dotato di tre schede di rete connette rispettivamente Internet, LAN interna e la DMZ in modo isolato.
  • DMZ a cuscinetto (Screened Subnet): La DMZ è isolata fisicamente interponendosi tra due firewall distinti posti in cascata (Firewall Esterno connesso a Internet e Firewall Interno connesso alla LAN), massimizzando la resilienza strutturale contro le intrusioni.
Syntax/Code Block: Cisco Extended Access Control List Setup 
! Configurazione di una ACL Estesa per proteggere la LAN aziendale
Router# configure terminal
Router(config)# access-list 101 permit tcp any host 192.168.2.10 eq 80
Router(config)# access-list 101 permit tcp any host 192.168.2.10 eq 443
Router(config)# access-list 101 permit tcp 192.168.1.0 0.0.0.255 any
Router(config)# access-list 101 deny ip any 192.168.1.0 0.0.0.255
Router(config)# access-list 101 permit ip any any

Router(config-if)# ip access-group 101 in
Router(config-if)# end

UdA n. 6: Reti Private Virtuali (VPN)

6.1 Architettura, Classificazione e Vantaggi delle VPN

Una VPN (Virtual Private Network) estende una rete privata locale oltre i confini geografici, sfruttando un'infrastruttura di trasporto pubblica e condivisa come Internet. Tramite protocolli crittografici, la VPN instaura un canale logico cifrato protetto definito Tunnel. Rispetto alle linee dedicate tradizionali (es. affitto di circuiti dedicati punto-punto fisici), le VPN offrono un abbattimento drastico dei costi di gestione aziendali e un'elevatissima scalabilità e flessibilità di configurazione degli endpoint.

Le VPN si classificano in base allo scenario operativo:

  • Remote-Access VPN: Consente ai singoli lavoratori remoti (es. in modalità Smart Working) di connettersi in modo sicuro alla rete aziendale centrale da qualsiasi postazione esterna tramite l'ausilio di un client software dedicato.
  • Site-to-Site VPN: Interconnette stabilmente intere sedi geograficamente distanti di una medesima azienda (es. filiale periferica e quartier generale), configurando i tunnel direttamente sui router perimetrali in modo del tutto trasparente per gli utenti interni.
Modello di Sicurezza Caratteristiche Strutturali dell'Infrastruttura VPN
Trusted VPN La sicurezza del percorso è garantita dall'operatore di telecomunicazioni che alloca circuiti dedicati esclusivi (es. MPLS), ma i dati non viaggiano necessariamente cifrati dal cliente.
Secure VPN I dati vengono cifrati, autenticati e validati all'origine tramite protocolli di sicurezza crittografici prima dell'inoltro sulla rete pubblica di transito.
Hybrid VPN Unisce le garanzie di instradamento di una rete Trusted con i protocolli di cifratura end-to-end di una rete Secure, massimizzando l'affidabilità aziendale.

6.2 Protocolli di Sicurezza per il Tunneling Crittografico

Per la realizzazione del tunnel cifrato si impiegano principalmente due suite protocollari:

  1. IPSec (Internet Protocol Security): Opera a livello 3 (Network). Suite robusta composta da due protocolli chiave: AH (Authentication Header) che garantisce solo l'integrità e l'autenticazione del pacchetto, ed ESP (Encapsulating Security Payload) che aggiunge la cifratura completa dei dati. IPSec può operare in Transport Mode (cifra solo il payload IP originale mantenendo intatta l'intestazione originaria) o in Tunnel Mode (l'intero pacchetto originario viene cifrato e incapsulato dentro un nuovo pacchetto IP esterno dotato di intestazioni inedite, ideale per connessioni Site-to-Site).
  2. SSL/TLS VPN: Opera a livello applicativo. Non richiede software client dedicati complessi poiché sfrutta l'architettura nativa dei comuni browser web, risultando ideale per accessi Remote-Access flessibili e granulari a servizi specifici.
Technical Vocabulary & CLIL Integration Tunneling: A method of transmitting data across a network by encapsulating one network protocol data unit within the payload of another packet.
Encapsulating Security Payload (ESP): A core IPSec protocol providing encryption, source data authentication, and integrity protection for network packets.

UdA n. 7: Reti Wireless e Reti Cellulari

7.1 Architettura e Componenti delle Reti Senza Fili

Le reti wireless eliminano la necessità del cablaggio fisico sfruttando onde radio o infrarossi. Si classificano in base al raggio di copertura geografica:

  • WPAN (Wireless Personal Area Network): Corto raggio (entro i 10 metri). Tecnologie chiave: Bluetooth (IEEE 802.15.1) adibito all'interconnessione di periferiche personali, e sistemi legacy IrDA a infrarossi.
  • WLAN (Wireless Local Area Network): Standard Wi-Fi (IEEE 802.11). I componenti base includono i Wireless Terminal (WT) e gli Access Point (AP) che collegano i client wireless alla rete cablata. La topologia si articola in BSS (Basic Service Set) gestita da un singolo AP, e ESS (Extended Service Set) formata dall'unione logica di più BSS interconnessi per consentire il roaming.
  • WMAN (Wireless Metropolitan Area Network): Copertura cittadina tramite standard WiMax (IEEE 802.16) per link point-to-multipoint a banda larga.
  • WWAN (Wireless Wide Area Network): Reti cellulari globali ad evoluzione generazionale (dall'architettura commutata del GSM/2G fino all'estrema densità cellulare e bassissima latenza del 5G).

7.2 Sicurezza, Autenticazione e Protocolli Wi-Fi

La natura aperta del mezzo radio espone intrinsecamente le reti wireless ad attacchi di intercettazione passiva (eavesdropping), inserimento di falsi access point (Rogue AP) e attacchi Man-in-the-Middle. L'evoluzione degli standard di sicurezza per il controllo accessi e la riservatezza comprende:

  1. WEP (Wired Equivalent Privacy): Basato sull'algoritmo a flusso RC4 con chiavi statiche corte e vulnerabilità nel vettore di inizializzazione (IV). Obsoleto e violabile in pochi minuti.
  2. WPA (Wi-Fi Protected Access): Introdotto come patch temporanea del WEP, implementa il protocollo TKIP per variare dinamicamente le chiavi crittografiche per ogni pacchetto, ma mantiene l'algoritmo RC4.
  3. WPA2: Standard robusto basato sull'algoritmo simmetrico avanzato AES operante in modalità CCMP. Ha garantito elevatissima sicurezza per anni, finché non è stato esposto l'attacco *KRACK* (Key Reinstallation Attack) mirato a forzare la reinstallazione delle chiavi durante il 4-way handshake.
  4. WPA3: Il paradigma moderno più sicuro. Sostituisce il sistema di handshake tradizionale con il protocollo SAE (Simultaneous Authentication of Equals), immune agli attacchi a dizionario offline e dotato di Forward Secrecy (la compromissione futura della password di rete non permette la decifratura del traffico registrato nel passato).
Modello di Autenticazione Centralizzata AAA (RADIUS) Nelle reti aziendali (Enterprise mode), la sicurezza degli accessi non si basa su una singola chiave condivisa pre-shared, bensì sull'architettura strutturata AAA (Authentication, Authorization, Accounting). L'Access Point funge da intermediario inoltrando le credenziali d'accesso fornite dall'utente a un server centrale dedicato operante tramite protocollo RADIUS (Remote Authentication Dial-In User Service). Il server convalida l'identità interrogando un database utenti protetto e traccia i tempi di connessione per finalità di auditing.

UdA n. 8: Virtualizzazione e Servizi Cloud

8.1 Tecniche e Architetture di Virtualizzazione

La Virtualizzazione consiste nell'interposizione di uno strato software astratto tra l'hardware fisico di una macchina e i sistemi operativi client. Questo strato software di controllo prende il nome di Hypervisor o VMM (Virtual Machine Monitor) e si suddivide in:

  • Hypervisor Tipo 1 (Bare-Metal): Viene eseguito direttamente sopra l'hardware fisico sottostante (es. VMware ESXi, KVM). Offre le massime prestazioni complessive, stabilità assoluta ed è lo standard dominante nei data center e nelle infrastrutture aziendali di produzione.
  • Hypervisor Tipo 2 (Hosted): Viene eseguito come applicazione software integrata all'interno di un sistema operativo ospitante tradizionale (es. VirtualBox, VMware Workstation). Utilizzato prevalentemente per scopi di testing e sviluppo software in ambienti isolati.

Il vantaggio primario della virtualizzazione risiede nel Consolidamento dei Server: ridurre drasticamente il numero di macchine fisiche necessarie, ottimizzando l'uso della CPU e della memoria (riducendo i tempi di inattività dell'hardware), abbattendo i consumi elettrici di raffreddamento e semplificando i piani di Disaster Recovery tramite snapshot immutabili dello stato di sistema.

8.2 Cloud Computing: Modelli di Servizio e Distribuzione

Il Cloud Computing rappresenta l'evoluzione naturale della virtualizzazione, consentendo l'accesso on-demand via Internet a risorse computazionali configurabili e scalabili, fornite secondo tre modelli di servizio principali:

Modello Cloud Definizione Estesa & Responsabilità di Gestione Esempi Pratici di Mercato
IaaS
(Infrastructure as a Service)		 Il provider alloca risorse hardware pure virtualizzate (CPU, storage di rete, macchine virtuali). L'utente ha il controllo totale del sistema operativo, delle applicazioni e delle configurazioni software. Amazon EC2, Google Compute Engine, Microsoft Azure VM.
PaaS
(Platform as a Service)		 Il provider fornisce la piattaforma e l'ambiente runtime ottimizzato per lo sviluppo e l'esecuzione del software. L'utente gestisce solo il proprio codice sorgente, dimenticando la manutenzione del sistema operativo sottostante. Heroku, AWS Elastic Beanstalk, Google App Engine.
SaaS
(Software as a Service)		 Il software applicativo finale è interamente ospitato, gestito e aggiornato dal provider cloud. L'utente finale vi accede direttamente tramite browser web senza installare alcuna componente locale. Google Workspace, Microsoft 365, Salesforce.

I modelli di distribuzione si articolano in Cloud Pubblico (risorse condivise multi-tenant fornite via Internet da provider terzi), Cloud Privato (infrastruttura dedicata ad uso esclusivo di una singola organizzazione aziendale) e Cloud Ibrido (integrazione orchestrata tra piattaforme pubbliche e private per consentire la migrazione flessibile dei carichi di lavoro più critici).

Technical Vocabulary & CLIL Integration Bare-Metal Hypervisor: A virtualization software installed directly on the server's hardware, offering near-native operational efficiency.
Multi-Tenancy: A cloud computing architecture where a single physical instance of a resource serves multiple distinct customers (tenants) while guaranteeing data isolation.

UdA n. 9: Preparazione all'Esame di Maturità ed Esercitazioni Critiche

9.1 Analisi Metodologica della Seconda Prova Scritta

Lo svolgimento ottimale del tema di Sistemi e Reti all'Esame di Stato richiede un approccio rigoroso e ingegneristico suddiviso in quattro fasi sequenziali vincolanti:

  1. Analisi del Testo e Scomposizione dei Requisiti: Isolare tutte le specifiche esplicite e implicite imposte dalla traccia ministeriale, distinguendo tra requisiti funzionali (es. numero di postazioni da servire, servizi pubblici da implementare) e requisiti non funzionali (es. vincoli di budget, tolleranza ai guasti, criteri di sicurezza).
  2. Scelta e Sviluppo dello Schema Logico di Rete: Disegnare l'infrastruttura identificando la dislocazione degli apparati (Core, Distribution e Access Layer). Pianificare l'adozione delle VLAN per segmentare i dipartimenti aziendali, e posizionare la DMZ e i firewall perimetrali.
  3. Pianificazione del Piano di Indirizzamento IP (Subnetting VLSM): Calcolare matematicamente le dimensioni delle sottoreti partendo dai requisiti più stringenti di host richiesti. Ottimizzare lo spazio degli indirizzi documentando il lavoro in una tabella riassuntiva pulita contenente l'intervallo di indirizzi validi, la subnet mask in notazione CIDR e il default gateway associato.
  4. Sviluppo degli Apparati Software e Scripting: Scrivere la sintassi di configurazione degli apparati di rete (Cisco IOS) per le interfacce, rotte statiche/dinamiche, DHCP, VLAN e liste di controllo degli accessi (ACL). Implementare gli script backend richiesti (PHP/PDO) e le query SQL avanzate di interfacciamento al database.
Modello Strutturato di Piano di Indirizzamento IP (Esempio Tema d'Esame) Avendo a disposizione la rete privata 172.16.0.0/16, si richiede il subnetting per tre dipartimenti aziendali:
- Subnet A (Produzione - 500 host richiesti): Richiede una maschera a 23 bit (29 - 2 = 510 host validi).
Indirizzo di Rete: 172.16.0.0/23 | Subnet Mask: 255.255.254.0 | Range IP validi: 172.16.0.1 - 172.16.1.254.
- Subnet B (Amministrazione - 200 host richiesti): Richiede una maschera a 24 bit (28 - 2 = 254 host validi).
Indirizzo di Rete: 172.16.2.0/24 | Subnet Mask: 255.255.255.0 | Range IP validi: 172.16.2.1 - 172.16.2.254.
- Subnet C (DMZ Server - 20 host richiesti): Richiede una maschera a 27 bit (25 - 2 = 30 host validi).
Indirizzo di Rete: 172.16.3.0/27 | Subnet Mask: 255.255.255.224 | Range IP validi: 172.16.3.1 - 172.16.3.30.
Syntax/Code Block: Query Avanzata SQL con Interrogazione Multitabella (JOIN & Aggregazione) 
-- Query d'esame: Estrazione del report dei log degli accessi sospetti in DMZ
-- Calcola il totale dei tentativi falliti per utente, filtrando solo quelli con media critica
SELECT 
    u.id_utente, 
    u.username, 
    COUNT(l.id_log) AS tentativi_falliti, 
    MAX(l.data_evento) AS ultimo_attacco
FROM utenti u
INNER JOIN log_accessi l ON u.id_utente = l.fk_utente
WHERE l.stato_autenticazione = 'FAILED' 
  AND l.data_evento >= DATE_SUB(NOW(), INTERVAL 30 DAY)
GROUP BY u.id_utente, u.username
HAVING COUNT(l.id_log) > 5
ORDER BY tentativi_falliti DESC;
