❓ Flashcard: Qual è la differenza logica strutturale sostanziale tra JSON e XML?

1.2 Formati di Scambio: JSON vs XML

Caratteristica XML (eXtensible Markup Language) JSON (JavaScript Object Notation)
Sintassi Struttura ad albero con tag di apertura/chiusura (<item>...</item>) Coppie chiave-valore e array ordinati ({"chiave": "valore"})
Parsing Richiede interpreti pesanti (DOM / SAX Parser) Immediato e nativo in JavaScript (JSON.parse())

Esempio di Laboratorio: Parsing XML DOM in Java (Gestione Magazzino/Spesa)

import javax.xml.parsers.DocumentBuilderFactory;
import javax.xml.parsers.DocumentBuilder;
import org.w3c.dom.Document;
import org.w3c.dom.NodeList;
import org.w3c.dom.Element;
import java.io.File;

public class MagazzinoParser {
    public static void main(String[] args) throws Exception {
        File file = new File("magazzino.xml");
        Document doc = DocumentBuilderFactory.newInstance().newDocumentBuilder().parse(file);
        doc.getDocumentElement().normalize();
        NodeList list = doc.getElementsByTagName("articolo");
        for (int i = 0; i < list.getLength(); i++) {
            Element elem = (Element) list.item(i);
            System.out.println("Prodotto: " + elem.getElementsByTagName("nome").item(0).getTextContent());
        }
    }
}

UdA 2: Generalità sulle Basi di Dati e Progettazione (Concettuale e Logica)

Modellazione dei dati aziendali dal modello concettuale alle forme normali

2.1 Architettura e Sicurezza DBMS

Un DBMS garantisce l'accesso controllato e concorrente ai dati salvaguardando tre pilastri sistemici fondamentali:

  • Privatezza e Controllo Accessi: Autenticazione e gestione dei privilegi utente (RBAC).
  • Integrità Fisica: Ripristino da guasti tramite meccanismi di Journaling e ACID log transazionali.
  • Consistenza: Transazioni atomiche che mantengono i vincoli logici stabili nel tempo.

2.2 Il Ciclo di Progettazione e Modello E/R

La progettazione concettuale formalizza la realtà tramite il diagramma Entità-Relazione (E/R) composto da:

  • Entità: Classi di oggetti reali (es. Biblioteca, Aeroporto, Turismo, Car Pooling).
  • Attributi: Proprietà delle entità (compreso l'identificatore/Chiave Primaria).
  • Associazioni e Cardinalità: Vincoli numerici strutturali d'interconnessione (0,1), (1,1), (0,N), (1,N).

2.3 Mappatura Logica e Normalizzazione

Regole di derivazione verso il Modello Relazionale:

  1. Ogni entità diventa una Relazione (Tabella).
  2. Le relazioni 1:N esportano la chiave primaria del lato 1 come Chiave Esterna (Foreign Key) nel lato N.
  3. Le relazioni N:N generano una nuova tabella ponte intermedia contenente le chiavi esterne ereditate.

Le Forme Normali contro le anomalie di ridondanza

  • 1FN: Attributi atomici ed esistenza di una Chiave Primaria univoca.
  • 2FN: In 1FN e ogni attributo non-chiave dipende totalmente dall'intera chiave primaria (no dipendenze parziali).
  • 3FN: In 2FN e nessun attributo non-chiave dipende transitivamente dalla chiave primaria tramite altri attributi non-chiave.
❓ Flashcard: Che differenza c'è tra WHERE e HAVING in SQL?
La clausola WHERE filtra i singoli record prima dell'esecuzione dell'aggregazione. La clausola HAVING filtra i gruppi generati dal GROUP BY dopo che le funzioni di aggregazione (SUM, COUNT, ecc.) sono state calcolate.

3.1 Algebra Relazionale

Linguaggio procedurale teorico basato su operatori:

3.2 Script SQL di Interrogazione e Controllo Avanzato

-- Creazione di una Vista Virtuale Logica
CREATE VIEW VistaIngegneria AS
SELECT id_studente, COUNT(id_esame) AS totale_esami, AVG(voto) AS media_voti
FROM esami
WHERE settore = 'INF'
GROUP BY id_studente
HAVING AVG(voto) >= 27;

-- Query con Subquery Annidata complessa (Uso di IN / ALL)
SELECT cognome, nome, email 
FROM studenti
WHERE id_studente IN (
    SELECT id_studente 
    FROM esami 
    WHERE voto > ALL (SELECT voto FROM esami WHERE id_corso = 12)
);

-- DCL: Gestione Sicurezza e Privilegi
GRANT SELECT, INSERT ON VistaIngegneria TO 'operatore_segreteria'@'localhost';
REVOKE DELETE ON studenti FROM 'operatore_segreteria'@'localhost';

-- Vincoli Fisici di Integrità Referenziale e Gestione Date
ALTER TABLE esami 
ADD CONSTRAINT fk_studenti
FOREIGN KEY (id_studente) REFERENCES studenti(id_studente)
ON DELETE CASCADE ON UPDATE CASCADE;

-- Esempio Manipolazione Temporale MySQL
SELECT id_studente, DATEDIFF(NOW(), data_immatricolazione) AS giorni_iscrizione
FROM studenti
WHERE YEAR(data_immatricolazione) = 2025;
❓ Flashcard: Come previene PDO gli attacchi di tipo SQL Injection?
PDO introduce i Prepared Statements (query preparate). Separando esplicitamente la struttura logica del comando SQL dai dati di input utente tramite segnaposto (es. :user), neutralizza l'interpretazione di codice malevolo inserito nei campi di input.

UdA 4: Programmazione Web Lato Server (PHP) e Interfacciamento Database

Sviluppo backend sicuro, persistenza dello stato e interfacciamento PDO

4.1 Fondamenti di Architettura PHP

PHP è un linguaggio di scripting interpretato lato server adibito alla composizione ipertestuale dinamica:

  • Scope: Variabili locali o globali. Per unire moduli riusabili si usano include (warning non bloccante) o require (blocco fatale esecutivo in caso di mancanza del modulo).
  • Strutture Dati: Array indicizzati numericamente e Array Associativi strutturati in coppie chiave-valore.
  • Interazione Stateless: I vettori globali $_GET e $_POST intercettano i flussi provenienti dai moduli HTML client-side. La sanitizzazione obbligatoria (es. htmlspecialchars()) neutralizza attacchi di tipo Cross-Site Scripting (XSS).

4.2 Script Pratico: Connessione PDO Sicura e Autenticazione (Hogwarts Login)

<?php
// Centralizzazione della connessione tramite file protetto (config.php)
$dsn = "mysql:host=localhost;dbname=Hogwarts;charset=utf8mb4";
$user = "root";
$pass = "password_sicura_2026";
$options = [
    PDO::ATTR_ERRMODE            => PDO::ERRMODE_EXCEPTION,
    PDO::ATTR_DEFAULT_FETCH_MODE => PDO::FETCH_ASSOC,
    PDO::ATTR_EMULATE_PREPARES   => false,
];

try {
    $pdo = new PDO($dsn, $user, $pass, $options);
} catch (\PDOException $e) {
    die("Errore di connessione al database: " . $e->getMessage());
}

// Controllo Login di Sicurezza con Prepared Statements
if ($_SERVER["REQUEST_METHOD"] === "POST") {
    $username = filter_input(INPUT_POST, 'user', FILTER_SANITIZE_SPECIAL_CHARS);
    $password = $_POST['pass'];

    $stmt = $pdo->prepare("SELECT id_utente, password_hash FROM utenti WHERE username = :username");
    $stmt->execute(['username' => $username]);
    $userRow = $stmt->fetch();

    if ($userRow && password_verify($password, $userRow['password_hash'])) {
        session_start();
        $_SESSION['auth'] = true;
        $_SESSION['user_id'] = $userRow['id_utente'];
        header("Location: dashboard.php");
        exit;
    }
}
?>

4.3 Persistenza dello Stato e Accessori Web

  • Cookie: Memorizzati sul browser del client ($_COOKIE), regolati da setcookie(). Vincolati alle normative europee GDPR (Cookie Banner).
  • Sessioni: Memorizzate in modo sicuro lato server ($_SESSION), legate all'identificativo effimero client-side PHPSESSID.
  • Upload File: Gestito dal vettore superglobale $_FILES combinato con verifiche di sicurezza sull'estensione MIME e la funzione move_uploaded_file().
❓ Flashcard: Qual è la principale differenza nell'overloading dei metodi tra Java e PHP?
Java supporta l'overloading nativo statico (più metodi con lo stesso nome ma firme/parametri diversi nello stesso perimetro). PHP non lo supporta nativamente e va emulato dinamicamente usando i metodi magici (es. __call) o parametri opzionali con valori di default.

❓ Flashcard: Translate the term "Integrità Referenziale" and its implications.
"Referential Integrity". It implies that any foreign key value must strictly match an existing primary key value in the parent relation, preventing orphan records within the transactional filesystem.

James Joyce: Asynchronous Streams

The modernist revolution and the Stream of Consciousness technique

Modernism and Subversion of Time

James Joyce tears apart the traditional Victorian structural timeline, mirroring the shift towards non-linear, asynchronous execution flows. The psychological interior monologue completely bypasses regular syntactic and grammatical boundaries.

"Eveline (from Dubliners, 1914): The protagonist is completely paralyzed by a psychological inability to escape her depressing life in Dublin. In 'Ulysses' (1922), the Stream of Consciousness reaches its peak, allowing thoughts, sensory reactions, and deep-seated memories to flow rawly and simultaneously, behaving just like an asynchronous data engine."
